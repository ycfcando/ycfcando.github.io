## JS执行上下文
1. 是什么？
   JS代码执行的环境，包含代码执行所需的内容。
2. 怎么做？
   1. 执行上下文会保存执行代码期间所用的变量、函数、this信息，执行代码期间会从其中进行取用。
   2. 执行上下文会在代码开始运行或函数调用时被创建，以在执行时被使用，分别被称为全局执行上下文和函数执行上下文。
   3. 创建的上下文会以一种结构实现变量、函数、this信息的存贮即：
      1. 变量对象：保存变量、函数。
      2. 作用域链：链表结构，首位为当前上下文的变量对象，之后依次为外层执行上下文的变量对象，它实现内层对外层数据的访问特性。
      3. this：当前执行代码所能访问的对象。全局执行上下文的this就是它的变量对象名为window。
   4. 创建的执行上下文会根据先后顺序以栈结构保存，且根据作用域链即上层能访问下层。
